### The Problem

In the unorganized sector of daily wage labor, exploitation, wage theft, and fraudulent claims are rampant due to a lack of verifiable employment history. Laborers often struggle to prove they worked at a specific site, while employers and contractors face disputes over unverified attendance. 

<br>

> RojgaarRecord solves this by introducing a transparent, immutable, and traceable employment ledger utilizing a strict 2-way assertion mechanism. This guarantees that every clock-in, clock-out, wage payment, and dispute resolution is cryptographically verified, mutually agreed upon, and indisputable.

<br><br>

### The Obscurity

To truly embrace the spirit of Stack Unknown, we rejected traditional relational databases, message brokers, and modern web frameworks. Instead, we architected the most delightfully absurd, yet perfectly functional tech stack imaginable:

<br>

* **The Database and Blockchain:** A live Minecraft Server. The database is a literal library of data constructed from Lecterns holding Written Books. The transaction history is literally a chain of blocks, with cryptographically hashed ledgers stored inside coloured Shulker Boxes arranged sequentially on physical iron chains across a custom-built plaza.

<br>

* **The Message Broker:** Autonomous backend integration using Discord bots and webhooks, with dedicated Discord channels acting as secure transaction pathways for the entire system.

<br>

* **The API Middleware:** A Python Flask daemon communicating with Minecraft via the archaic RCON protocol.

<br>

* **The Frontend UI and Rendering Engine:** Google Sheets. We stripped away traditional web apps and turned Google Sheets into a dynamic rendering engine. The spreadsheet acts purely as a Read-Only Display Canvas, dynamically generating native charts and tables, controlled exclusively by brutalist HTML sidebars acting as the Remote Control.

<br>

> The platform transforms Minecraft into a highly secure, visual database while turning Google Sheets into a powerful dynamic view controller.

<br><br>

### Tech Stack

* **Minecraft (Paper 1.20.1):** The immutable ledger and physical data visualization layer.
* **Python 3.12:** API Server, automated simulation orchestrators, and cryptographic hashing.
* **Google Apps Script:** The backend controller dictating the entire ERP flow.
* **Google Sheets:** The dynamic data visualization canvas.
* **Discord API:** Real-time SMS simulation, autonomous bot integration, and transaction queue pathways.
* **Vanilla HTML, CSS, and JS:** Brutalist, zero-dependency sidebars embedded in Sheets.

<br><br>

### Final Implementation Features

1. **Physical Blockchain Ledger:** 8 distinct transaction zones physically rendered in Minecraft, featuring real-time cryptographic validation and tamper-evident hashes.

<br>

2. **2-Way Assertion Claims:** A comprehensive dispute resolution system where workers submit claims, contractors accept or reject, and employers execute final overrides.

<br>

3. **Role-Based ERP Sidebars:** Dedicated brutalist UIs for Workers, Contractors, Employers, and Admins embedded directly into Google Sheets.

<br>

4. **Dynamic Sheet Rendering:** Google Sheets automatically wipes and re-renders customized, professional read-only views on the fly based on sidebar interactions.

<br>

5. **Simulated SMS OTP Verification:** Secure, simulated multi-factor authentication routing through Discord channels to verify physical presence for daily wage payouts.
